# GreyOrange Warehouse Automation Project Management

This project provides a full end‑to‑end workflow for a Project Manager overseeing
large‑scale warehouse automation deployments for **GreyOrange**. GreyOrange’s
Relay Pick solution leverages a hybrid fleet of **vertical tote movers (VTMs)** and
**horizontal tote movers (HTMs)**. VTMs move totes within aisles and rearrange them
within storage bays, while HTMs shuttle totes from cantilevers to work stations at
high speed (13 ft/s) and automatically lift totes to an ergonomic height
【628455820589393†L147-L156】. Using VTMs and HTMs together maximises vertical space
utilisation and increases throughput, enabling high‑density storage (up to five
totes deep) and up to **400 tote presentations per hour**【628455820589393†L112-L117】【628455820589393†L166-L173】. The
GreyMatter warehouse execution system (WES) orchestrates these robots alongside other
technologies to reduce fulfilment cost per unit and improve peak‑season performance
【797358567226614†L131-L134】.

The synthetic dataset and analysis in this repository allow a Project Manager to explore
how various factors—such as warehouse size, robot mix, risk level and tool usage—influence
project duration, cost and commissioning performance. While the data are simulated,
the patterns mimic realistic GreyOrange deployment scenarios, making the project
useful for training, benchmarking or preparing for real implementations.

## Contents

- **`generate_pm_dataset.py`** – Script to generate a synthetic dataset of warehouse
  automation projects. Each row represents a project with fields such as region,
  warehouse size, number of HTMs and VTMs, team size, complexity and risk categories,
  estimated and actual duration/costs, tool usage (VOS, CAD, Tower), commissioning days
  and derived variances. Use this script to create a new dataset each time you run it.
- **`pm_project_dataset.csv`** – Sample dataset generated by the script with 300
  projects.
- **`analysis_pm_robotics.py`** – Analysis script that answers five key business
  questions by computing descriptive statistics, running simple linear regressions
  and producing diagnostic plots (optional). The questions cover complexity
  effects, robot mix, tool utilisation, risk categories and regional differences.
- **`pm_tools.xlsx`** – Excel workbook listing typical tools (VOS, CAD, Tower, HTM,
  VTM) and installation tasks required for warehouse automation projects.
- **Environment files** – `requirements.txt` and `environment.yml` describe the
  Python dependencies needed to run the scripts.

## Synthetic Data Generation

The data generation script creates a reproducible dataset of GreyOrange projects. It
simulates realistic variability across regions (North America, Europe, Asia Pacific,
Latin America), warehouse sizes (50 k–500 k sq ft), numbers of HTMs and VTMs, team
sizes, complexity levels and risk categories. Estimated duration and cost are
computed based on project scale and risk; actual duration and cost deviate from
estimates according to complexity and risk. Tool usage flags indicate whether
planning tools such as the **Visualisation and Optimisation Suite (VOS)**,
**Computer‑Aided Design (CAD)** or **Tower** stacking software were used. These tools
help improve layout design and reduce commissioning time. Commissioning days,
schedule variance and cost variance are derived fields.

To generate a new dataset with 400 projects, run:

```bash
python generate_pm_dataset.py --output my_projects.csv --n_projects 400 --seed 123
```

## Business Questions Addressed

The analysis script explores five key questions:

1. **Impact of project complexity on schedule and cost variance** – Do larger
   warehouses with more robots tend to overrun schedules and budgets?
2. **Effect of robot mix (HTMs vs. VTMs)** – Does a higher proportion of VTMs
   relative to HTMs influence project performance or commissioning time?
3. **Tool utilisation** – How do planning tools (VOS, CAD, Tower) correlate
   with schedule adherence and commissioning days?
4. **Risk category influence** – Which risk categories (Low, Medium, High) are
   associated with greater deviations from estimates?
5. **Regional differences** – Are there performance disparities across regions
   (e.g., North America vs. Asia Pacific)?

Each question is answered by summarising the relevant metrics and, where
appropriate, fitting a simple regression model. Diagnostic plots are optional and
saved to a `figures` directory when the `--show_plots` flag is used.

## Running the Analysis

1. **Set up your environment:**
   - With conda:
     ```bash
     conda env create -f environment.yml
     conda activate greyorange_pm_env
     ```
   - Or using `pip`:
     ```bash
     python -m venv venv
     source venv/bin/activate
     pip install -r requirements.txt
     ```
2. **Generate a dataset (optional):**
   ```bash
   python generate_pm_dataset.py --output pm_project_dataset.csv --n_projects 300
   ```
3. **Run the analysis:**
   ```bash
   python analysis_pm_robotics.py --input pm_project_dataset.csv --show_plots
   ```
   Remove the `--show_plots` flag if you only want textual output. Plots will be saved in a
   `figures` directory. Note that generating plots requires Matplotlib and may take a few
   seconds.

## Sample Findings

Running the analysis on the provided sample dataset yields the following insights:

- **Complexity matters:** Projects categorised as **High complexity** (larger warehouses
  and more robots) have average schedule overruns of ~15 days and higher cost
  variances. Regression shows both warehouse size and total robots contribute
  positively to schedule and cost variance.
- **Robot mix:** Projects with a higher proportion of VTMs do not show a
  strong difference in variance compared to more HTM‑heavy projects, suggesting that
  robot mix alone is not a dominant driver.
- **Tool utilisation:** Projects that used VOS and CAD exhibited higher variances in the sample
  dataset, but they also tended to be larger, more complex projects. Tower usage showed
  slightly reduced cost variance and commissioning time, hinting that Tower may
  improve efficiency.
- **Risk category:** High‑risk projects experienced greater schedule and cost
  deviations, whereas medium‑risk projects performed slightly better.
- **Regional differences:** Average durations and costs varied modestly across regions,
  with Europe showing slightly lower commissioning times.

These patterns are illustrative; they should be validated with real project data.

## Next Steps

1. **Incorporate real data:** Replace or complement the synthetic dataset with anonymised
   historical project data from GreyOrange deployments to validate the patterns.
2. **Advanced modelling:** Apply more sophisticated models (e.g., random forests,
   gradient boosting) to predict schedule and cost variance based on project attributes.
3. **Cost‑benefit analysis of tools:** Conduct controlled experiments to measure
   the impact of VOS, CAD and Tower on commissioning speed and budget adherence.
4. **Expand feature set:** Include additional variables such as vendor partnerships,
   workforce experience, or seasonal factors.
5. **Visual dashboards:** Build interactive dashboards (e.g., using Plotly or Tableau)
   to allow project managers to explore scenarios and track live project performance.
